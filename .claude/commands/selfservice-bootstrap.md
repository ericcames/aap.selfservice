# AAP Self-Service Bootstrap

This skill bootstraps a fresh RHDP environment for the self-service portal demo. It collects credentials, generates a named inventory, then runs all three playbooks in sequence: `bootstrap_aap.yml` (configure AAP), `bootstrap_portal.yml` (deploy portal on OpenShift), and `sync_portal_orgs.yml` (patch portal configmap with correct orgs, sync frequency, and permissions).

See [`.claude/commands/references/context.md`](references/context.md) for object names and API endpoints.

When complete, the AAP instance is configured and the portal is running and usable. Run `/selfservice-first-time` first if this is a new machine.

## Preflight Check

```bash
# ansible.cfg with Hub token
grep -q "galaxy_server.rh_certified" ~/.ansible/ansible.cfg 2>/dev/null && \
  grep -v "PASTE_YOUR_TOKEN_HERE" ~/.ansible/ansible.cfg | grep -q "token=" && \
  echo "✅ ansible.cfg" || echo "❌ ansible.cfg"

# collections
test -d ~/.ansible/collections/ansible_collections/ansible/platform && \
  echo "✅ ansible.platform" || echo "❌ ansible.platform"
test -d ~/.ansible/collections/ansible_collections/ansible/controller && \
  echo "✅ ansible.controller" || echo "❌ ansible.controller"
test -d ~/.ansible/collections/ansible_collections/kubernetes/core && \
  echo "✅ kubernetes.core" || echo "❌ kubernetes.core"

# oc CLI
oc whoami 2>/dev/null && echo "✅ oc — logged in" || echo "❌ oc — not logged in"

# helm CLI
helm version --short 2>/dev/null && echo "✅ helm" || echo "❌ helm"
```

If any check fails, stop and tell the user:
```
❌ Prerequisites missing: <list each failing item>

Run /selfservice-first-time to set up these prerequisites before bootstrapping.
```

## Step 1 — Identify the Environment

Ask for two values to name the inventory:
- **Customer account** — short name (e.g. `acme`, `ames`, `ibm`)
- **Planned demo** — short name (e.g. `develop`, `f5demo`, `linuxdemo`)

Inventory will be created at: `inventories/rhdp-<customer>-<demo>/`

Check if this inventory already exists:
```bash
ls inventories/rhdp-<customer>-<demo>/ 2>/dev/null && echo "exists" || echo "new"
```

If it exists, ask whether to reuse it or overwrite it.

## Step 2 — Resolve AAP Credentials

### 2a. Environment variables (highest priority)
```bash
echo "CONTROLLER_HOST=${CONTROLLER_HOST}"
echo "CONTROLLER_USERNAME=${CONTROLLER_USERNAME}"
echo "CONTROLLER_PASSWORD=${CONTROLLER_PASSWORD}"
```

If all three are non-empty, skip to Step 3.

### 2b. Prompt the user
Ask for:
- **AAP URL** — from the RHDP Services page (e.g. `https://aap-aap.apps.cluster-xxxx.dynamic.redhatworkshops.io`)
- **AAP Password** — from the RHDP Services page (username defaults to `admin`)

Tell the user to set these in their shell:
```bash
export CONTROLLER_HOST=<url>
export CONTROLLER_USERNAME=admin
export CONTROLLER_PASSWORD=<password>
```

## Step 3 — Resolve OCP Credentials

### 3a. Environment variables
```bash
echo "OCP_HOST=${OCP_HOST}"
echo "OCP_TOKEN=${OCP_TOKEN}"
```

If both are non-empty, skip to Step 4. Also verify `oc` is logged in to the correct cluster:
```bash
oc whoami --show-server
```

### 3b. Prompt the user
Ask for:
- **OCP API URL** — from the RHDP Services page (e.g. `https://api.cluster-xxxx.dynamic.redhatworkshops.io:6443`)
- **OCP Token** — from the RHDP Services page

Tell the user to set these and log in:
```bash
export OCP_HOST=<ocp_api_url>
export OCP_TOKEN=<token>
oc login $OCP_HOST --token=$OCP_TOKEN --insecure-skip-tls-verify=true
```

## Step 4 — Generate the Inventory

Create the inventory:

**`inventories/rhdp-<customer>-<demo>/hosts`:**
```ini
[all]
localhost ansible_connection=local ansible_python_interpreter=/usr/bin/python3
```

**`inventories/rhdp-<customer>-<demo>/group_vars/all.yml`:**
```yaml
---
# RHDP environment: rhdp-<customer>-<demo>
# Generated by /selfservice-bootstrap on <date>
aap_hostname: "{{ lookup('ansible.builtin.env', 'CONTROLLER_HOST') }}"
aap_username: "{{ lookup('ansible.builtin.env', 'CONTROLLER_USERNAME') }}"
aap_password: "{{ lookup('ansible.builtin.env', 'CONTROLLER_PASSWORD') }}"
aap_validate_certs: false
ocp_host: "{{ lookup('ansible.builtin.env', 'OCP_HOST') }}"
ocp_token: "{{ lookup('ansible.builtin.env', 'OCP_TOKEN') }}"
hub_token: "{{ lookup('ini', 'token section=galaxy_server.rh_certified file=~/.ansible/ansible.cfg') }}"
hub_auth_url: "https://sso.redhat.com/auth/realms/redhat-external/protocol/openid-connect/token"
portal_namespace: aap-portal
```

Remind the user that `group_vars/all.yml` contains no secrets — all sensitive values resolve at runtime via lookups.

## Step 5 — Run bootstrap_aap.yml

Show this command and ask for confirmation before running:

```bash
ansible-playbook -i inventories/rhdp-<customer>-<demo>/ playbooks/bootstrap_aap.yml```

### Interpreting output

| Result | Meaning |
|--------|---------|
| `ok` | Object already exists and is correct — not a warning |
| `changed` | Object was created or updated |
| `failed` | Real error — stop and report |

A mix of `ok` and `changed` on a re-run is expected and correct.

## Step 6 — Run bootstrap_portal.yml

Show this command and ask for confirmation before running:

```bash
ansible-playbook -i inventories/rhdp-<customer>-<demo>/ playbooks/bootstrap_portal.yml```

## Step 6a — Run sync_portal_orgs.yml

This step is required after every portal deploy. The Helm chart sets three defaults that break the portal for non-admin users:
- `orgs: Default` only — users in other AAP orgs can't log in
- `permission.enabled: true` with no policies — RBAC denies all non-admins
- Sync frequency `minutes: 60` — catalog is empty for an hour after deploy

`sync_portal_orgs.yml` patches all three in one configmap update. Show this command and run it (no confirmation needed — it is safe and idempotent):

```bash
ansible-playbook -i inventories/rhdp-<customer>-<demo>/ playbooks/sync_portal_orgs.yml```

Expected output includes:
- `"Configuring portal to allow users from: [...]"` — lists all AAP orgs found
- `changed` on the configmap patch and deployment restart tasks
- `"Portal updated. Users in these AAP organizations can now log in: [...]"`

## Step 7 — Verify Results

Create a session token:
```bash
TOKEN_JSON=$(curl -s -k -X POST \
  -H "Content-Type: application/json" \
  -u $CONTROLLER_USERNAME:$CONTROLLER_PASSWORD \
  "$CONTROLLER_HOST/api/gateway/v1/tokens/" \
  -d '{"description":"bootstrap-verify token","scope":"write"}')
TOKEN=$(echo $TOKEN_JSON | python3 -c "import sys,json; print(json.load(sys.stdin)['token'])")
TOKEN_ID=$(echo $TOKEN_JSON | python3 -c "import sys,json; print(json.load(sys.stdin)['id'])")
```

Verify AAP objects:
```bash
# Hub certified
curl -s -k -H "Authorization: Bearer $TOKEN" \
  "$CONTROLLER_HOST/api/controller/v2/credentials/?name=Automation+Hub+-+certified" \
  | python3 -c "import sys,json; c=json.load(sys.stdin)['count']; print('✅ Automation Hub - certified' if c>0 else '❌ Automation Hub - certified — NOT FOUND')"

# Hub validated
curl -s -k -H "Authorization: Bearer $TOKEN" \
  "$CONTROLLER_HOST/api/controller/v2/credentials/?name=Automation+Hub+-+validated" \
  | python3 -c "import sys,json; c=json.load(sys.stdin)['count']; print('✅ Automation Hub - validated' if c>0 else '❌ Automation Hub - validated — NOT FOUND')"

# Project
curl -s -k -H "Authorization: Bearer $TOKEN" \
  "$CONTROLLER_HOST/api/controller/v2/projects/?name=aap.selfservice" \
  | python3 -c "
import sys,json
results=json.load(sys.stdin)['results']
if results:
    status=results[0]['status']
    print(f'{'✅' if status=='successful' else '⚠️'} aap.selfservice (sync: {status})')
else:
    print('❌ aap.selfservice — NOT FOUND')"

# OAuth application — lives in the GATEWAY registry, not the controller
curl -s -k -H "Authorization: Bearer $TOKEN" \
  "$CONTROLLER_HOST/api/gateway/v1/applications/?name=aap-selfservice-portal" \
  | python3 -c "import sys,json; c=json.load(sys.stdin)['count']; print('✅ aap-selfservice-portal (OAuth app)' if c>0 else '❌ aap-selfservice-portal — NOT FOUND')"
```

Verify portal in OCP:
```bash
oc get pods -n aap-portal 2>/dev/null | grep -v "^NAME" | \
  python3 -c "
import sys
pods = [l for l in sys.stdin.read().splitlines() if l.strip()]
if not pods:
    print('❌ aap-portal namespace — no pods found')
else:
    for p in pods:
        fields = p.split()
        name, ready, status = fields[0], fields[1], fields[2]
        icon = '✅' if status == 'Running' else '❌'
        print(f'{icon} {name} ({status}, ready: {ready})')"
```

Delete the verification token:
```bash
curl -s -k -X DELETE \
  -H "Authorization: Bearer $TOKEN" \
  "$CONTROLLER_HOST/api/gateway/v1/tokens/$TOKEN_ID/"
```

Print the final summary:
```
Bootstrap complete.

  AAP:
    ✅ Automation Hub - certified
    ✅ Automation Hub - validated
    ✅ aap.selfservice (sync: successful)
    ✅ aap-selfservice-portal (OAuth app)

  OpenShift (aap-portal namespace):
    ✅ <pod name> (Running, ready: 1/1)

Portal URL: https://rhaap-portal-aap-portal.apps.<cluster>
```

If any object shows ❌, tell the user which object is missing and suggest re-running `/selfservice-bootstrap` to retry.

### On failure

Show the error from the playbook output and suggest the most likely fix based on the error message.

If the user cannot reach the OCP API or AAP URL directly (connection timeout, TLS errors that suggest a network path issue), ask for bastion credentials and offer to walk through connecting via the bastion:

```
The RHDP Ansible Product Demo always includes a bastion host with oc CLI pre-configured.
If your laptop can't reach the cluster directly, you can run the playbooks from the bastion.

Get from RHDP Services page:
  - Bastion SSH command (e.g. ssh lab-user@ssh.ocpv05.rhdp.net -p <port>)
  - Bastion password
```
